Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Section elements are semantic elements which define a section in a document with thematic content
  • Semantic elements have names which describe its content and its role within a webpage. Semantic elements make a webpage more accesible and easier to read for other developers and also the browser
  • Anything within the head element will not be displayed for the end user, it contains metadata for use by the browser

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A period designates a term as a class selector. Any HTML element assigned to the stated class, e.g. .card would gain the property assignments from the CSS
  • There are three main methods to incorporate CSS into a html webpage, inline CSS which styles an element directly in the HTML file, an internal style sheet which contains css rules for the webpage in the head section of the HTML file, the rules only apply to that page. Lastly, there is the external CSS style sheet, this is preferred in most cases as it allows developers to keep all their CSS rules in a separate file.

Git

  • git status: checks what branch we are currently on
  • git checkout: switches to specified branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: Pulls the latest version of the code from git, specifically for this example it will pull the main code
  • git push origin main: pushes the current version of the code we are working on from the local system, again this will be the main code [dont use main lol]
